使用Hexo+GitHub搭建个人博客（二）

  • Hexo

写在前面

上一篇我们已经搭建好了自己的个人博客，现在我们来讲讲里面的一些配置，美化我们的博客和增强博客的功能。

配置网站基本信息

打开我们hexo工程目录下的_config.yml文件，可以看到一大堆配置内容，不过我们需要注意的就下面几项
title: 网站标题
subtitle: 副标题
description: 个人签名
author: 姓名
language: zh-Hans
这几项是我们需要修改的，可以根据自己的喜好改(language: zh-Hans 是中文)

修改网站主题

如果不喜欢hexo默认的主题，我们可以在github上搜索hexo theme，会有很多主题
进入详情页都有介绍，预览图，我们可以下载自己喜欢的主题后解压到hexo目录下的themes文件夹中
然后打开hexo工程目录下的_config.yml文件，修改theme为我们下载的主题（本站主题为spfk）

更新网站

修改完网站基本信息和主题后，在hexo目录下右键Git Bash Here，重新输入
hexo clean && hexo g && hexo d
这样会在public文件夹中重新我们需要上传的最新的文件
然后把public里的所有文件重新复制到我们本地clone下来的github仓库（覆盖原有文件）
接下来打开GitHub for windows，发现Changes里面会有那些覆盖之后发生改变的文件
最后commit to master提交本地后点击Fetch origin按钮同步到github即可
打开浏览器入username.github.io，我们的个人博客果然变了

绑定域名

首先我们需要买一个域名，腾讯云，阿里云等都可以。
拿到域名之后，进行解析，推荐使用dnspod免费解析。
进入网站注册登录后，点击域名解析选项，添加好域名后，增加俩条记录，如下图。

然后登录你注册域名网站的管理后台，修改DNS解析地址为
f1g1ns1.dnspod.net
f1g1ns2.dnspod.net
最后登录github，找到项目，点击Settings，找到GitHub Pages，输入域名，Save即可

如果延迟的话，可能解析立马不会生效，需要等会儿才能看到。
